# Screenshots

This directory is intended to hold screenshots of original tool output produced during the lab exercises.

## What to Add Here

Add screenshots of **your own tool output only**. Examples include:

- `nmap-scan-results.png` — Output from Nmap host discovery and port scanning commands
- `nmap-service-enum.png` — Service version enumeration results (`-sV` scan)
- `nbtscan-output.png` — NetBIOS device discovery results
- `gvm-dashboard.png` — GVM / Greenbone Security Assistant dashboard showing scan tasks
- `gvm-vulnerability-report.png` — GVM vulnerability scan results with severity breakdown
- `metasploit-session.png` — Metasploit console output showing successful exploitation (sanitized if needed)
- `simplerisk-risk-register.png` — SimpleRisk risk register view
- `simplerisk-risk-score.png` — Risk scoring and prioritization view in SimpleRisk
- `ftp-anonymous-access.png` — FTP service misconfiguration demonstration

## Naming Convention

Use lowercase, hyphen-separated names that clearly describe the content:

```
<tool>-<what-it-shows>.png
```

Examples:
- `nmap-open-ports.png`
- `gvm-critical-findings.png`
- `metasploit-vsftpd-exploit.png`

## Important Copyright Notice

> **Only add screenshots of your own tool output.**  
> Do **not** upload screenshots of the Jones & Bartlett Learning lab platform, lab instructions, or any JBL proprietary interface. All content in this directory must be original output generated by you during your lab work.

## Embedding Screenshots in Lab Summaries

Once added, reference screenshots in the relevant lab summary files like this:

```markdown

![Nmap Scan Results](../Screenshots/nmap-open-ports.png)
```
